# VERSION 1 - 开发安排说明（同步 `refer.txt`）

## 1. V1 目标
- 在 3 分钟限时内完成循环玩法：`输入图案 -> 自动填充 -> 消除判定 -> 计分 -> 下一图案`。
- 支持基础棋盘 `8*8`，并使用 `N*N` 二维数组表达（初始全 0）。
- 支持 `T/I/O/L/J/S/Z` 图案库（含已定义旋转形态）。

## 2. V1 规则落地（已同步最新需求）

### 2.1 棋盘定义
- 棋盘数据结构：`board: number[][]`。
- 默认尺寸：`8*8`，保留参数化能力（`N*N`）。
- 初始值：全部为 `0`，填充后对应格子置为 `1`。

### 2.2 图案放置规则（核心）
- 图案由 `0/1` 矩阵表示，`1` 代表占用格。
- 自动寻位顺序：
  1. 先筛选“纵向高度满足”的候选位置。
  2. 在候选位置中优先“尽量靠上”。
  3. 同一高度下按“从左到右”选取首个可放位置。
  4. 成功后一次性写入棋盘。
- `refer.txt` 中 5x5 示例应作为单元测试样例固化。

### 2.3 消除规则
- 单次图案填充后统一判定。
- 行满或列满均触发消除。
- 若同时消除多行/列，记本次总消除次数为 `M`。

### 2.4 计分规则
- 单次得分：`score = M*20 + (M-1)*70`。
- 边界约束：`M=0` 时本次得分强制为 `0`（不允许负分）。
- 总分为每次提交得分累加。

## 3. 开发安排（按优先级执行）

### P0：规则与核心逻辑（必须先完成）
1. **规则冻结文档**
   - 把“尽量靠上 + 从左到右”写成明确判定逻辑。
   - 用示例描述同一图案在不同棋盘状态下的落点结果。
2. **核心数据模型**
   - `Board`、`Shape`、`GameState`、`Placement` 四类结构统一。
3. **核心算法函数**
   - `findPlacement(board, shape)`
   - `placeShape(board, shape, placement)`
   - `clearLines(board): M`
   - `calcScore(M): number`
   - `applyTurn(state, shape): state`
4. **测试优先**
   - 针对放置、消除、计分建立单元测试。
   - 必含 `refer.txt` 的填充示例测试。

### P1：可玩闭环（先能跑完整一局）
1. 倒计时系统（3 分钟）。
2. 最小可视化界面（棋盘、当前图案、分数、时间）。
3. 一局流程跑通（开始、进行、结束结算）。

### P2：体验与调优
1. 填充/消除/加分反馈。
2. 图案出现节奏与难度调优。
3. 文案提示与结算信息完善。

## 4. 任务拆分与验收标准

### 任务 A：放置算法
- **完成标准**
  - 满足“先靠上，再从左到右”。
  - 冲突检测正确，不越界。
  - 至少 10 条单测通过（含边界用例）。

### 任务 B：消除与计分
- **完成标准**
  - 可正确统计同时消行/消列次数 `M`。
  - `M=0/1/2/3` 计分正确。
  - 消除后棋盘状态与预期一致。

### 任务 C：3 分钟玩法闭环
- **完成标准**
  - 可完整游玩一局并产生总分。
  - 时间归零后禁止继续提交并展示结算。

## 5. 当前版本约束（V1 不做）
- 暂不做复杂动画与音效系统。
- 暂不做联网、排行、账号体系。
- 暂不扩展新图案类型与特殊道具。
