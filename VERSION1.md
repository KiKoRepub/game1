# 游戏需求分析与开发优先级（基于 `refer.txt`）

## 一、当前描述中需要完成的内容

### 1) 核心玩法目标
- 在限时 3 分钟内，玩家通过寻找并提交图案来获得尽可能高的分数。
- 图案由不定数量的 `1*1` 方块组成，存在多种形状与旋转形态。
- 图案成功提交后，需要自动填充到预设棋盘中，并触发消除与计分逻辑。

### 2) 棋盘与填充规则
- 基础棋盘大小为 `8*8`（文中也提到通用 `N*N`，建议先支持 8*8，再抽象到 N*N）。
- 填充规则需要明确实现：
  - 优先考虑“纵向高度满足”的可放置位置；
  - 在满足条件的位置中，按“从左到右”顺序寻找第一个可放位置；
  - 找到后执行填充。
- 该规则本质上是一个“自动落位/自动摆放”算法，需要可复现、可测试。

### 3) 消除规则
- 当某一整行或某一整列被填满时，触发消除。
- 一次图案填充后，可能触发多次消除（记为 `M` 次）。

### 4) 计分规则
- 单次提交得分公式：
  - `score = M*20 + (M-1)*70`
- 需要约定 `M=0` 的得分（通常应为 0，避免出现负分）。
- 需要把“单次得分”累加到“总分”。

### 5) 图案库内容（版本 1）
- 已给出 7 类基础图案及旋转形态：
  - `T / I / O / L / J / S / Z`
- 这些图案应整理为统一数据结构，供“识别”“放置”“预览”等模块共用。

### 6) 待补充的实现细节（当前描述未完全定义）
- 玩家“找到图案”的输入方式：点击？框选？拖拽？拍照识别？
- 一次操作的循环流程：找到后是否立即入板？是否可撤销？
- 新图案出现机制：随机池？权重？是否可预览下一个？
- 游戏结束条件：仅时间到结束，还是还要考虑“无可放位置”提前结束？
- 消除时机：单次填充后统一判定，还是边填边判定。
- 动画与反馈：填充、消除、连消、加分飘字、音效等。

## 二、建议的分阶段开发路径（逐步完成）

### 阶段 0：规则澄清（必须先完成）
先把会影响代码架构的规则写死，避免返工：
- 明确“纵向高度满足 + 从左到右”的精确定义（可举 3~5 个样例）。
- 明确 `M=0` 时得分为 0。
- 明确一次提交能否同时消行消列、是否允许连锁（二次坍塌）。
- 明确游戏循环：输入 -> 放置 -> 消除 -> 计分 -> 下一次输入。

> 产出：1 份“规则说明 v1”，附示例。

### 阶段 1：纯逻辑最小闭环（无 UI 或极简 UI）
- 实现数据模型：
  - 棋盘 `board`（8*8）
  - 图案 `shape`（二维 0/1 数组）
  - 游戏状态（剩余时间、总分、回合）
- 实现核心函数：
  1. `findPlacement(board, shape)`：按规则找位置
  2. `placeShape(board, shape, pos)`：填充
  3. `clearLines(board)`：统计并清除满行/满列，返回 `M`
  4. `calcScore(M)`：按公式计算
  5. `tickGameState(...)`：完成一次完整提交流程
- 为以上函数写单元测试（重点测边界和冲突场景）。

> 目标：先保证“规则正确”，再做表现层。

### 阶段 2：可玩原型（先能玩）
- 接入最小交互界面（哪怕是简陋网格）。
- 展示：
  - 当前棋盘状态
  - 当前图案/待提交图案
  - 当前分数、倒计时
- 支持用户完成完整 3 分钟一局。

> 目标：验证玩法是否成立，能不能持续产生决策和乐趣。

### 阶段 3：体验完善
- 动画反馈（填充、消除、加分）
- 声音反馈
- 文案与提示（连消提示、剩余时间警告）
- 新手引导与失败/结算面板

### 阶段 4：平衡与内容扩展
- 调整图案出现概率与节奏（难度曲线）。
- 评估计分公式是否过于奖励高连消，必要时微调。
- 增加模式：无尽模式、挑战模式、每日关卡等。

## 三、优先要解决的问题（按优先级排序）

### P0（最优先，决定项目能否推进）
1. **放置规则精确定义**  
   “纵向高度满足”是当前最关键但最容易歧义的规则，必须先定清楚。
2. **单次操作流程定义**  
   一次提交后，消除判定与计分顺序必须固定，直接影响核心算法。
3. **核心数据结构统一**  
   图案、棋盘、坐标系统（行列定义）先统一，避免后续模块接口不一致。

### P1（规则稳定后立即做）
1. **核心算法与测试**  
   放置、消除、计分三件套 + 单元测试。
2. **时间系统与结算**  
   倒计时、结束判定、总分结算。
3. **基础可玩 UI**  
   让规则“看得见、玩得到”。

### P2（可玩后提升质量）
1. 交互反馈和动画
2. 参数调优（难度、分数增长速度）
3. 内容扩展与留存功能
